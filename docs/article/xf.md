### 前端埋点系统
>
> 为了能够在用户反馈之前察觉到一些线上问题，减少线上故障率和提高纠错能力，团队内部孵化自研一套埋点系统

**主要功能：** 前端埋点sdk开发，并将sdk引入到各个项目中使用；服务端收集sdk发送的代码异常、资源加载异常等数据，将其整理汇总入库；后台系统根据业务需要，将这些数据按照不同产品、不同分类、标记状态进行展示，并进行相应的数据报表展示

**服务端技术栈：** koa + mysql2 + sequelize + redis + log4js

**后台系统技术栈：** vue2.x + element-ui + axios + ts + echarts

**工作职责：** 协同负责部分服务端接口处理、后台系统部分业务开发以及独立负责sdk埋点开发

`项目亮点`

- 对每个异常错误有标记状态，出现次数达到某个阀值，会有预警通知
- 除了统计错误信息外，还会采集每台设备打开项目的加载和白屏时间，通过这些可以对项目有一个很好的评估

`项目可改进点：`

- 项目可增加用户行为统计，可以更好对用户行为作出分析并根据这些分析，来进行业务产品的调整
